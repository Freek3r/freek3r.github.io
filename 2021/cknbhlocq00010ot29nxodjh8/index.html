什么是SQL注入

SQL注入是比较常见的网络攻击方式之一，它不是利用操作系统的漏洞来实现攻击，而是针对程序员编写时的疏忽，通过SQL语句，实现无账号登录，甚至篡改数据库。

SQL注入步骤

以 http://www.example.com/index.php?id=1 为例，SQL注入无非就是想尽办法获取数据，而获取数据需要知道每个数据的表、字段以及库名，为了知道这三个信息，我们需要从 MySQL的information_schema 或者MSSQ了的 Master 里获取。

综上所述，无论是联合查询注入(SQL_EXPRESS UNION SELECT SQL_EXPRESS_2)、堆叠注入(SQL_EXPRESS_1; SQL_EXPRESS_2)或者盲注(IF(EXPRESS1, SLEEP(N), NULL)) ，最终的目的还是获取库名、表名和字段名然后再进行其他操作。

下面只展示通用的模板，但是任何注入都基本上八九不离十、换汤不换药，遇到具体的情况具体分析即可。

判断是否能注入

整数型：id=1 => 正常，id=1' => 报错，id=2-1 => 正常且显示 id=1 的页面

字符型：id=1 => 正常，id=1' => 报错，id=1'# => 正常且显示与 id=1 相似的页面

获取列数

id=1' ORDER BY x，从 1 开始尝试，一直到 x ，如果 x 报错，而 x-1 不报错，则说明有 x-1 个列

获取数据库

SELECT schema_name FROM information_schema.schemata

获取表

SELECT table_name FROM information_schema.tables WHERE table_schema=库名

获取字段

SELECT column_name FROM information_schema.columns WHERE table_schema=库名 AND table_name=表名
